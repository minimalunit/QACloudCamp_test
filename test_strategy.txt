В рамках тестирования нового функционала будем проводить следующие виды тестирования:
1. Смоук-тестирование для проверки готовности нового функционала для дальнейшего тестирования. Проверяем что облачный сервис работает, что форма нового функционала появилась в веб-интерфейсе, базовые функции работают (ID тест-кейсов 1, 2). 
2. Функциональное тестирование, в рамках которого проверяем проверяем работоспособность всего нового функционала, валидацию всех полей ввода.  return Применяем техники граничных значений и классов эквивалентности для проверки каждого поля ввода (ID тест-кейсов 3-7).  return В том числе проверяем элементы из юзабилити тестирования, что новой функциональностью удобно пользоваться конечному пользователю (ID тест-кейсов 14, 15).
3. Регрессионное тестирование, проверяем что введение новых функций не внесло изменения в функционирование системы (ID тест-кейсов 8, 9).
4. Кроссбраузерность и кроссплатформенность.  return Проверяем интерфейс формы во всех указанных в требованиях браузерах, если таковых нет, то запрашиваем статистику пользователей облачного сервиса, чтобы определить браузеры для тестирования.  return В случае отсутствия статистики, подбираем браузеры по браузерным движкам в соответствии со статистикой из открытых источников данных.   return Проверяем и десктопные, и мобильные варианты браузеров на разных операционных системах.   return Проверяем различные разрешения экрана. Можно применить технику попарного тестирования для уменьшения количества проверок. (ID тест-кейсов 10-13)
5. Тестирование производительности.   return Проверяем стабильность и надежность нового функционала, стрессовое тестирование, изменение времени выполнения операций в зависимости от загрузки.
